✨ Números aleatorios en Python · breve historia

Desde sus primeras versiones, Python incorporó la generación de números aleatorios a través del módulo random, basado en el algoritmo Mersenne Twister. Este generador pseudoaleatorio, creado en 1997 por Makoto Matsumoto y Takuji Nishimura, ofrecía un período enorme de 2¹⁹⁹³⁷−1 y una distribución uniforme de alta calidad, convirtiéndose rápidamente en la herramienta estándar para simulaciones, juegos y pruebas estadísticas. La comunidad adoptó random por su sencillez y velocidad, aunque siempre se advirtió que no era apto para fines criptográficos.

Con el auge de la ciencia de datos y el machine learning, el ecosistema científico exigió generación vectorizada y distribuciones más especializadas. Así, NumPy incorporó su propio subsistema numpy.random, primero basado en el mismo Mersenne Twister pero optimizado para arreglos multidimensionales. Más adelante, en 2019, NumPy introdujo un nuevo generador de números aleatorios (default_rng) con algoritmos más flexibles (PCG64 por defecto) y un diseño moderno que permite reproducibilidad y mejor rendimiento. Paralelamente, el módulo secrets llegó en Python 3.6 para proporcionar aleatoriedad criptográficamente segura, usando fuentes dependientes del sistema operativo.

Hoy, Python ofrece un ecosistema completo de aleatoriedad: random para tareas cotidianas, secrets para tokens y contraseñas, y numpy.random para grandes volúmenes de datos y distribuciones estadísticas. Esta evolución refleja la madurez del lenguaje: cada herramienta está diseñada para un nicho específico, desde simples lanzamientos de dados hasta simulaciones cuánticas, manteniendo el equilibrio entre velocidad, reproducibilidad y seguridad. La historia de los números aleatorios en Python es también la historia de cómo una comunidad convirtió una necesidad básica en un conjunto de soluciones de talla mundial.

# A y B deben tener exactamente la misma forma if A.shape == B.shape: suma = A + B resta = A - B else: print("Dimensiones diferentes: no se puede sumar/restar")
✖️ MULTIPLICACIÓN MATRICIAL (A @ B)
# Válido si A.shape[1] == B.shape[0] (nº columnas de A = filas de B) if A.shape[1] == B.shape[0]: producto = A @ B # también np.dot(A, B) o np.matmul(A,B) print(producto) else: print("Nº columnas de A debe coincidir con filas de B")
🔢 MULTIPLICACIÓN ESCALAR (cualquier tamaño)
escalar = 5.0 resultado = escalar * A # broadcasting: funciona siempre
🔄 TRANSPUESTA (cualquier tamaño)
A_trans = A.T # también np.transpose(A)
📐 DETERMINANTE (solo matrices cuadradas)
if A.shape[0] == A.shape[1]: det = np.linalg.det(A) else: print("La matriz no es cuadrada: no se puede calcular determinante")
↩️ INVERSA (cuadrada y no singular)
if A.shape[0] == A.shape[1]: try: inv = np.linalg.inv(A) except np.linalg.LinAlgError: print("Matriz singular (no invertible)") else: print("Se requiere matriz cuadrada")

# Ejemplo: matriz (3,1) + vector fila (3,) -> numpy extiende dimensiones # No siempre funciona para suma/resta si las formas no son compatibles. # Verificar broadcasting: (m,n) con (n,) puede funcionar, (m,n) con (k,n) requiere k=m.
